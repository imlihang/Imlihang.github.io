Member template 成员模板

鲫鱼是鱼类的派生类，麻雀是鸟类的派生类。

由鲫鱼和麻雀组成的 pair 可以初始化成 鱼类和鸟类的 pair，反之则不行。

成员模板使得构造函数更加有弹性。

标准库的写法，智能指针是一个类。

普通指针中基类的指针可以指向派生类，智能指针也要满足这个写法。

所以在构造函数过程中利用了成员模板。

specialization 模板特化

一般泛化

特化，当调用 cout << hash()(1000)；时，会调用特化的版本。

partial specialization 模板偏特化 –个数的偏

partial specialization 模板偏特化 –范围的偏

偏特化由 任意类型 变为 指向任意类型的指针。

template template parameter 模板模板参数

Container 用到 T 作为模板参数

这不是 template template parameter

C++11

variadic templates(since C++11)

模板函数会递归调用 print 将 args 一个个输出。

当调用输出最后一个参数后，包的参数为 0，会调用参数为空的 print

auto (since C++11)

ranged-base for (since C++11)

reference

面向对象三种关系

对象模型 vptr 和 vtbl

函数调用方法

静态绑定: call xxx

动态绑定: 符合三个条件，1. 指针调用，2. 向上转型 up cast，3.调用虚函数 编译器会动态绑定

容器中如果放不同形状的对象，做不到。因为容器要放大小相同的数据。

所以容器中放了指针，通过虚函数调用不同的 draw 函数。

如果要用 C 的写法，就需要每个形状做判断，如果是圆形调用圆形的 draw ，如果是矩形调用矩形的 draw。

这个写法很繁琐，而且如果之后要增加子类，就要改变判断的代码。

对象模型 关于 this

子类对象调用父类函数，this 指针符合三个条件，会转成动态绑定的形式。

对象模型 关于 Dynamic Binding

谈谈 const

标准库的 string 底层是共享一个字符串内存

如果要修改某一个非常量字符串，设计者需要考虑再申请一块字符串空间，在新的空间上修改。

如果是常量字符串，设计者就不需要考虑 copy on write。函数比较简洁，比较快。

关于 new, delete

重载 ::operator new, ::operator delete,::operator new[], ::operator delete[]

重载的是全局的 new delete

重载 member operator new/delete

重载 member operator new[]/delete[]

示例

在 new array obj 的时候，会多出四个字节来存储 array_size

:: new/delete 可以跳过重载的 new/delete 来使用全局的 new/delete

重载 new(),delete()

标准库 string 中，为了多申请一些内存来实现引用计数，用了 placement new 设计。
