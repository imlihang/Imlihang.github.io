7.栈 堆 和 内存管理

stack 存在于某作用域(scope)的内存空间

heap/system heap ，指操作系统提供的一块 global 内存空间，程序可动态分配从某种取得若干区域。

stack objects 的生命周期

c1 便是 stack object，其生命在作用域 (scope)结束时结束。 这种作用域內的 object，又成为 auto object，因为它会被自动清理。

static local objects 的生命期

c2 便是 static object，其生命在作用域 (scope) 结束之后仍然存在，直到整个程序结束。

heap objects 的生命期

P 所指的便是 heap object，其生命 在它被 delete 之时结束。

以上出现內存泄露 (memory leak)，因为当作用域结束，p 所指的 heap object 仍然存在，但指针 p 的生命却结束了，作用域之外再也看不到 p (也就沒机会 delete p）

new：先分配 memory, 再調用 ctor

delete：先調用 dtor, 再釋放 memory

9.复合，继承，委托

复合Composition 表示 has-a

设计模式 Adapter

queue 几乎没有自己的实现，复用了 deque 的函数。这种方式称为Adapter

从内存的角度去看

复合关系下的构造和析构

构造由內而外

Container 的构造函数首先调用 Component 的 default 构造函数，然后才执行自己。

编译器会默认帮你加默认构造函数的调用，如果要调用含参的构造函数，要自己写。

析构由外而内

Container 的析构函数首先执行自己，然后才调用 Component 的析构函数。

Delegation（委托） Composition by reference.

成员中有一个对象的引用。

设计模式 Handle/Body(pImpl)

外部接口，对外接口可以不变，内部实现可以调整，内部实现不影响外部接口。

生命周期与复合那种方式不一样。

复合是同时生成同时销毁。

委托可以不同时生成。

可以有很多对象，内部共享一个实现。

引用计数，当其中一个对象要改变，就要 new 一份新的对象出来。copy on write

Inheritance 继承 表示 is-a

Inheritance 继承 关系下的构造和析构

base class 的 dtor 必须是 virtual， 否则会出现 undefined behavior

构造由内而外

Derived 的构造函数首先调用Base的default构造函数， 然后才执行自己。

Derived::Derived(…): Base() { };

析构由外而内

Derived 的析构函数首先执行自己，然后才调用 Base 的析构函数。

Derived::~Derived() { …… ~Base()};

Inheritance 继承 with virtual functions(虚函数)

non-virtual 函数：你不希望 derived class 重新定义 （override， 覆写） 它.
virtual 函数：你希望 derived class 重新定义 （override， 覆写） 它，且你对它已有默认定义。
pure virtual 函数：你希望 derived class 一定要重新定义 （override 覆写）它，你对它没有默认定义。

设计模式 Template Method

当框架设计的时候，父类可以先写好已经确定的部分，留出不确定的部分让子类实现。

如打开文件的方式可以让子类实现，其它确认的逻辑可以写好。MFC 基本是这样实现

Delegation(委托) + Inheritance(继承)

设计模式 Observer

解决问题，数据更新后，该数据的多个负责可视化的对象会同时更新。

设计模式 Composite

例子：假如说一个文件系统，一个目录可以包含一个文件，一个目录又可以包含一个目录。

设计模式 Prototype

问题：创建未来未知 class 对象。
